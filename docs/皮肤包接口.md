# 皮肤包接口 v1

皮肤包只替换画面，不改变动作状态机、行顺序、触发条件、播放速度、随机权重或交互逻辑。

## 官方外置皮肤

```text
小泰妍.exe
skins/
  taeyeon-invu/
    skin.xml
    frames.zip
```

`skin.xml`：

```xml
<?xml version="1.0" encoding="utf-8"?>
<skin apiVersion="1"
      id="taeyeon-invu"
      name="INVU 月影女神"
      developer="个人非商业开发"
      exclusiveAction="月影侧身行礼"
      archive="frames.zip" />
```

约束：

- 文件夹名必须与 `id` 完全一致；`id` 只能包含英文字母、数字、点、下划线和连字符。
- `apiVersion` 固定为 `1`，`archive` 固定为 `frames.zip`。
- `name`、`developer` 和可选 `exclusiveAction` 最长 64 个字符。
- `exclusiveAction` 非空时启用 r15，属性值就是菜单名称；它不能改写代码、触发条件或其他行。
- 本地只保存所选皮肤 ID；皮肤缺失或损坏时自动回退内置“Weekend 粉蓝周末”。

## 帧归档

`frames.zip` 根目录直接包含：

```text
r00/c00.png
r00/c01.png
...
r23/c07.png
motion/r00/c00-r00-c01.mtn
...
```

不得额外包一层 `frames/`。每张图必须是带透明通道的 `528×808` PNG。`motion/` 是构建脚本生成的紧凑双向运动网格，不是图片。归档必须只包含约定帧和合法网格，不能包含绝对路径、`..`、反斜杠路径、重复条目或其他文件。

总计必须为 176 张关键帧。旧皮肤不写 `exclusiveAction` 仍有效，r15 只作兼容占位，不出现在动作菜单或随机待机。启用专属动作时，r15 必须包含 8 张属于该皮肤、姿势连续的关键帧。r16、r17 必须继续提供合法占位；r18–r23 是所有皮肤共享的公共动作。

拖拽与自动漫游不触发 r01/r02。r22 和 r23 的进入、保持/循环和点击退出语义由程序控制，皮肤不得调整其区间。

## 运行时换肤

合规皮肤依次经过“人物 alpha 切片伪 3D 旋身 → 新造型由头到脚显露 → r21/c07 最终 pose → r00 待机”。完整过渡固定为 60 个显示阶段。候选归档先在独立缓存中完成安全校验和关键帧预热，过渡结束时才原子替换；连续选择时只保留最后一次选择。

## 运行时插值

皮肤保存 176 张关键帧，可附带官方脚本生成的 `motion/*.mtn`。运行时在相邻关键帧间形变单一不透明姿态，使完整循环至少包含 24 个显示阶段，并沿用固定基线总时长。看鼠标的 16 个方向即时切换，不做循环补间。不得加入 `c08.png` 等额外帧。

## 安全上限

- 单个 ZIP 条目不超过 32 MiB。
- `frames.zip` 不超过 128 MiB。
- 全部条目解压后总量不超过 128 MiB。
- 任何清单、路径、重复条目、大小、尺寸、透明度或网格校验失败，只淘汰该外置皮肤，不影响程序启动和内置皮肤。
